
# PBCLA Directory Overview

The `PBCLA` directory contains tools related to peptide bond cleavage analysis. It is mainly used for MGF file processing, peptide fragment bond annotation, and data format conversion.

## File Overview

- `pbcla.py`:
  - Core implementation of the PBCLA algorithm. It takes an input MGF file, computes peptide fragment bond counts, missing fragment bonds, and related information, then writes the results back to an output MGF file.
  - Example command:
    ```bash
    python pbcla.py --in_mgf_path ./mgf_dataset/example.mgf --out_mgf_path ./mgf_dataset/example_out.mgf
    ```

- `mgf2csv.dbond_m.py`:
  - Converts an MGF file processed by PBCLA into the CSV format required for dbond_m training and evaluation, including multi-label information.
  - Example command:
    ```bash
    python mgf2csv.dbond_m.py --mgf_path ./mgf_dataset/example_out.mgf --csv_path ./mgf_dataset/example.multi.csv
    ```

- `mgf2csv.dbond_s.py`:
  - Converts an MGF file processed by PBCLA into the CSV format required for dbond_s training and evaluation, including both single-label and multi-label information.
  - Example command:
    ```bash
    python mgf2csv.dbond_s.py --mgf_path ./mgf_dataset/example_out.mgf --csv_path ./mgf_dataset/example.csv
    ```

- `utils.py`:
  - Provides helper utilities for the PBCLA algorithm, including peptide objects, mass threshold calculations, and binary search utilities.

- `mgf_dataset/`:
  - Stores example MGF files and converted outputs such as `example.mgf`, `example_out.mgf`, `example.csv`, and `example.multi.csv`.

## CSV Field Description

The CSV files generated by PBCLA are used for downstream classification tasks. The input and output fields are listed below.

### Input Fields

- `seq`: amino acid sequence of the mirror-image protein, stored as a string. For example, `AEFDE` represents a sequence containing 6 amino acids and therefore 5 peptide bonds.
- `charge`: precursor ion charge, integer type. For example, `2` means the ion carries 2 positive charges.
- `pep_mass` or `m_z`: mass-to-charge ratio of the precursor ion, floating-point type. For example, `544.6621`.
- `intensity`: signal intensity, floating-point type. For example, `116622.119`.
- `nce`: normalized collision energy, integer type. For example, `30` indicates a collision energy of 30 eV.
- `scan_num`: scan index of the spectrum, integer type. For example, `219` means the 219th scan.
- `rt`: retention time, floating-point type.
- `bond_aa` (`dbond_s`): the dipeptide amino acid fragment corresponding to a peptide bond, such as `AE`.
- `bond_pos` (`dbond_s`): the position of the peptide bond in the sequence, starting from 0.

### Output Fields

- `bond_label` (`dbond_s`): cleavage state of a single peptide bond. `0` means not cleaved, and `1` means cleaved.
- `true_multi` (`dbond_m`): multi-label cleavage sequence, such as `1;0;1;0;1`, where each position represents the cleavage state of one peptide bond.
- `mb`: positions of missing fragment bonds, separated by semicolons. For example, `1;3;5;` means the 1st, 3rd, and 5th peptide bonds are missing.
- `tb`: total number of peptide bonds.
- `fb`: number of matched peptide bonds, that is, cleaved peptide bonds.
- `fbr`: fragmented bond ratio, defined as `fbr = fb / tb`, representing the cleavage ratio of peptide bonds.

#### Output Example

For the sequence `AEFDE`, a cleavage sequence like `[1,0,1,0,1]` means:

- the 1st peptide bond (`AE`) is cleaved
- the 2nd peptide bond (`EF`) is not cleaved
- the 3rd peptide bond (`FD`) is cleaved
- the 4th peptide bond (`DE`) is not cleaved
- the 5th peptide bond (`ED`) is cleaved

## Data Processing Pipeline

A typical data processing workflow is as follows:

1. Convert raw tandem mass spectrometry data to MGF format.
   For example, RAW files can be converted to MGF files with MSConvert.
2. Annotate the MGF files.
   Use `pbcla.py` to annotate peptide bond cleavage information in the raw MGF files and generate MGF files containing fragment annotations.
3. Convert MGF files to CSV format.
   Use `mgf2csv.dbond_m.py` or `mgf2csv.dbond_s.py` to convert the annotated MGF files into CSV files.
4. Train and evaluate downstream models.
   Use the generated CSV data for model training and evaluation.

If you need custom parameters or batch processing, refer to the function calls inside the scripts.
